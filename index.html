Final State Architecture

The final state architecture for the Toronto Data Platform is designed to empower data-driven decision-making, enable seamless data sharing across divisions, and support advanced analytics with scalability and reliability.

This diagram highlights the scalable, reliable, and user-friendly architecture of the Toronto Data Platform.

Key Components

  • Amazon S3: Central data lake for storing Raw, Aggregated, Curated and Extended datasets.
  • AWS Glue: Facilitates ETL processes, data transformation, and cataloging.
  • Amazon Redshift: Managed data warehouse for analytics and reporting.
  • Amazon Kinesis: Real-time data streaming for ingestion and analysis.
  • Amazon SageMaker: Supports ML model development, training, and deployment.
  • AWS Lambda: Serverless data processing.
  • IAM: Secure access control and governance for resources.
  • Amazon DataZone: Simplifies data discovery, sharing, and governance across teams.
  • AWS Organizations: Centralized management of multiple AWS accounts for governance and security.
  • CloudFormation: Infrastructure-as-code service to automate AWS resource provisioning.
  • Iceberg Table: Open table format for high-performance analytical queries on large datasets.
  • Databricks: Unified analytics platform for data engineering, machine learning, and collaboration.
  • PySpark: Python-based API for distributed data processing and machine learning with Spark.
  • & much more...

To learn more about the approaches and methodologies used to measure and track success metrics, click the button below to expand the key insights section.

Key Insights

  • Data Freshness: Measured using pipeline monitoring tools like Apache Airflow and AWS CloudWatch to ensure data is updated within acceptable timeframes.
  • Validation Error Rates: Tracked through automated data validation processes and error reporting tools to maintain data accuracy and reliability.
  • Time-to-Data: Calculated from API usage logs and query execution times to ensure timely access to data for stakeholders.
  • Value Generation: Assessed by measuring ROI improvements and cost savings derived from data initiatives.
  • User Adoption: Monitored using engagement metrics, survey responses, and usage statistics from various data products and APIs.
  • Compliance: Evaluated by conducting regular audits and using automated compliance checks against regulatory frameworks.
  • Data Governance Maturity: Measured through adherence to governance best practices, such as policies defined by DAMA-DMBOK, and periodic reviews of data hygiene practices.

The success metrics framework ensures a comprehensive approach to track technical and business performance. By focusing on these metrics, we can drive alignment across divisions, measure progress, and achieve long-term organizational goals.
